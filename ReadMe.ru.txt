Технические требования:
Разработайте и внедрите REST API с использованием Hibernate/Spring/SpringMVC (предпочтительно Spring-Boot!) без внешнего интерфейса.

Задача:
Создайте систему голосования, чтобы решить, где пообедать.

1. 2 типа пользователей: администратор и обычные пользователи
2. Администратор может ввести:
2.1. ресторан
2.2. и его обеденное меню дня (обычно 2-5 пунктов, только название блюда и цена)
2.2.1. Меню меняется каждый день (обновления делают администраторы).
3. Пользователи могут проголосовать за ресторан, в котором они хотят сегодня пообедать.
3.1. Учитывается только один голос от пользователя
3.2. Если пользователь проголосует снова в тот же день:
3.2.1. Если до 11:00, мы предполагаем, что он передумал.
3.2.2. Если после 11:00, то уже поздно, голосовать нельзя.
4. Каждый ресторан предлагает новое меню каждый день.

5. В результате предоставьте ссылку на репозиторий github.
5.1. Он должен содержать код, README.md с документацией по API
5.2. и пару команд curl для его проверки (лучше — ссылку на Swagger).

6. P.S.: Убедитесь, что все работает с последней версией, которая есть на github :)
7. P.P.S.: Предположи, что ваш API будет использоваться фронтэнд-разработчиком для создания фронтэнд-интерфейса поверх него.
